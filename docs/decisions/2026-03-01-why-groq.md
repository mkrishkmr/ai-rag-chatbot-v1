# Architecture Decision Record: Groq (Llama-3.1-8b-instant)

## Context
The application relies heavily on real-time text generation (Server-Sent Events streaming) to drive a modern, terminal-like Chat UI inside Next.js. Any perceptible delay between the user hitting "Send" and the first token appearing degrades the user experience.

## Alternatives Considered
- **OpenAI GPT-4o-mini**: Standard fallback, decent latency.
- **Google Gemini 1.5 Flash**: Good multimodal capability.
- **Groq LPU hosting Llama-3.1-8b-instant**: specialized hardware.

## Decision
We selected **Groq with Llama-3.1-8b-instant**.

## Justification
- **Incredible Latency**: Groq uses Language Processing Units (LPUs) that routinely spit out several hundred tokens per second. Time To First Token (TTFT) is minimal.
- **Strict Adherence**: The `llama-3.1-8b` variant has demonstrated exceptional capability to rigidly adhere to our extensive `SYSTEM_PROMPT` rules (e.g. `RULE 4 — NO ADVICE`, `RULE 14 — NO URLS IN ANSWER`).
- **Open Source Leverage**: It allows us to pivot to our own hosting in the future if data localization requirements tighten for Indian FinTech data.
